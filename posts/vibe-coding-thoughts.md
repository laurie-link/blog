最近一直在用 AI 开发自动化工具，提高工作效率。需要的功能这两天已基本完善，来做下这段时间氛围编程的经验总结

结果上：原来我可能要花 80%左右精力在重复性、低效劳动上，20%花在真正能提高自己价值，学到东西的劳动。现在有了工具，重复性劳动时间大大缩短，上班心情都好很多

## 如何提高氛围编程效率的经验

### 起码要懂基本语法

首先，虽然氛围编程对我们自身编程技能要求极低，但基本语法多少还是要懂一点，争取至少能看懂代码，这样我们才能更好地指挥 AI

### 及时清理屎山

“屎山代码”就是没用的代码。我们让 AI 添加各种功能时，AI 难免会添加不需要的功能，导致后期项目中出现很多屎山代码

现在的 AI 靠 RAG 技术的语义搜索项目代码来了解项目，然后实现我们想要的功能。当项目中出现大量屎山，可能会干扰 RAG 结果，误导 AI 不说，AI 甚至可能去改屎山代码，而不是我们真正需要改的地方。从我的经验来看，屎山代码是氛围编程项目后期难以推进的主要原因

所以及时清理屎山很重要。这时候大家可能会想，不懂代码怎么清理？我的经验是，清理屎山正是学习代码的重要过程。因为平时懒得学，只有屎山多的导致项目推进不下去，才不得不逼自己看代码

### 把握好需求的尺度

这一点我想表达的是，我们应该把“能用”放在第一位，并且不急于追求“美化”和“易用”。因为在我们不懂代码的情况下，激进地追求“美化”、“易用”很可能增加与 AI 间无意义的拉扯，特别是 AI 在“美化”、改善易用性过程中把先前完善的功能破坏时

正确实践，按我的经验看，应该是在自己对代码底层逻辑有把握，心里明确知道怎么指挥 AI 的基础上，再对外观，易用性做进阶优化。对底层逻辑的把握是靠上面两点得到

## 氛围编程的常用工具：

### 1\. cursor

最近收费收紧，新的计费方式下，20 美元才用几天就会限额，感觉很恶心。前两个星期发邮件给官方，要求给我换回旧的 500 次对话次数计费，但是说我那个账号是计费政策更换之后创建的，不给换。然后我用我旧账号充的，现阶段使用旧计费政策使用中

除此之外，使用方面，个人感觉 cursor 仍然是最好用的氛围编程 IDE。不说质量，cursor 的 Claude-4-sonnet 一口气写 2000 行代码都没问题，对比 Claude code 使用十次左右就要等待几小时，起码使用流畅度很不错。况且质量上和 Claude code 差的不是很远

如果你有 cursor 旧账号，cursor 依然是数一数二的选择

### 2\. Claude Code

Anthropic 新推出的终端编程工具。看来最好的东西人家还是自己留着，Claude code 的编码能力确实是 T0 级别。一些在 cursor 里不好解决的问题，交给 Claude code 或许能解决

除上面说的使用量限制，Claude code 就是比较容易被封号。但人家比较良心，封号会退款

### 3\. Trae

Trae 虽然编码能力跟上面两个有可感知的差距，但人家性价比挺高。新用户只要 3 美元就能对话六百次

我一般清理屎山、学习代码这种就会用 Trae，感觉它很适合干这个活哈哈哈

另外，Trae 还在研发 Solo 模式啥的，含金量在上升

### 4\. 谷歌云白嫖 Gemini API Key

这个用来学习代码不错，Gemini 2.5 Pro 老师每次讲代码讲的很好。但这 API Key 获取难度非常高
